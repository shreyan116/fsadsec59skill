📄 2. Ticket.java (Entity class)

JPA annotated entity — fields match the table columns. Uses javax.persistence (compatible with Hibernate 5).

package com.klef.fsad.exam;

import javax.persistence.Entity;
import javax.persistence.Id;

@Entity
public class Ticket
{
    @Id
    private int id;                 // primary key
    private String name;             // ticket description / title
    private String date;             // ISO format (e.g. 2026-03-12)
    private String status;           // Booked, Confirmed, Cancelled, etc.

    public Ticket() {}               // mandatory no-arg constructor

    // ---------- getters / setters ----------
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getDate() { return date; }
    public void setDate(String date) { this.date = date; }

    public String getStatus() { return status; }
    public void setStatus(String status) { this.status = status; }
}

Note The date is stored as String for simplicity; in production you may use java.time.LocalDate + converter.

⚙️ 3. hibernate.cfg.xml (essential configuration)

Placed in src/main/resources. Adjust URL, credentials, dialect according to your DB.

<!DOCTYPE hibernate-configuration PUBLIC
        "-//Hibernate/Hibernate Configuration DTD 3.0//EN"
        "http://www.hibernate.org/dtd/hibernate-configuration-3.0.dtd">
<hibernate-configuration>
    <session-factory>
        <!-- Database connection (MySQL example) -->
        <property name="hibernate.connection.driver_class">com.mysql.cj.jdbc.Driver</property>
        <property name="hibernate.connection.url">jdbc:mysql://localhost:3306/fsadexam</property>
        <property name="hibernate.connection.username">root</property>
        <property name="hibernate.connection.password">yourpassword</property>

        <!-- Hibernate properties -->
        <property name="hibernate.dialect">org.hibernate.dialect.MySQL8Dialect</property>
        <property name="hibernate.show_sql">true</property>
        <property name="hibernate.format_sql">true</property>
        <property name="hibernate.hbm2ddl.auto">update</property>  <!-- auto-create tables -->

        <!-- Entity mapping -->
        <mapping class="com.klef.fsad.exam.Ticket"/>
    </session-factory>
</hibernate-configuration>

▶️ 4. ClientDemo.java — HQL insert & update

Inserts a new ticket using session.save(), then performs an HQL update with positional parameters. Important: HQL uses bean property names, not table columns. The query updates fields name and status where id matches.

package com.klef.fsad.exam;

import org.hibernate.Session;
import org.hibernate.SessionFactory;
import org.hibernate.Transaction;
import org.hibernate.cfg.Configuration;
import org.hibernate.query.Query;          // Hibernate 5+ Query interface

public class ClientDemo
{
    public static void main(String[] args)
    {
        // 1. Build SessionFactory from hibernate.cfg.xml
        Configuration cfg = new Configuration();
        cfg.configure("hibernate.cfg.xml");      // reads configuration
        SessionFactory sf = cfg.buildSessionFactory();

        // 2. First session: insert a record
        Session session1 = sf.openSession();
        Transaction tx1 = session1.beginTransaction();

        Ticket t = new Ticket();
        t.setId(1);
        t.setName("Movie Ticket");
        t.setDate("2026-03-12");
        t.setStatus("Booked");

        session1.save(t);                         // insert into Ticket table
        tx1.commit();
        System.out.println("✅ Ticket Inserted (ID: 1)");
        session1.close();

        // 3. Second session: HQL update
        Session session2 = sf.openSession();
        Transaction tx2 = session2.beginTransaction();

        // HQL update using positional parameters (?1, ?2, ?3)
        String hql = "update Ticket set name=?1, status=?2 where id=?3";

        Query q = session2.createQuery(hql);
        q.setParameter(1, "Flight Ticket");       // ?1 → name
        q.setParameter(2, "Confirmed");           // ?2 → status
        q.setParameter(3, 1);                      // ?3 → id = 1

        int result = q.executeUpdate();            // executes the update
        System.out.println("🔁 " + result + " record updated (HQL)");

        tx2.commit();
        session2.close();
        sf.close();
    }
}

Console output (expected):

Hibernate: insert into Ticket (date, name, status, id) values (?, ?, ?, ?)
✅ Ticket Inserted (ID: 1)
Hibernate: update Ticket set name=?, status=? where id=?
🔁 1 record updated (HQL)
⚠️ HQL parameter style: ?1, ?2 (indexed parameters) works in Hibernate 5. For Hibernate 6+ prefer :name style, but the code above is compatible with Hibernate 5.x.

🧪 6. How to run & verify

  1. Create a MySQL database named fsadexam.
  2. Update hibernate.cfg.xml with your DB credentials.
  3. Run mvn clean compile exec:java -Dexec.mainClass="com.klef.fsad.exam.ClientDemo" (or run from IDE).
  4. Check the table Ticket after execution: row with id=1 should have name = "Flight Ticket", status = "Confirmed".

💡 Verification SQL:

SELECT * FROM Ticket WHERE id = 1;
-- result: 1, 'Flight Ticket', '2026-03-12', 'Confirmed'

📚 Additional HQL examples (for practice)

Other common HQL operations you can add to ClientDemo:

// SELECT query (after insert)
String hqlSelect = "from Ticket where status = :st";
Query<Ticket> selectQ = session.createQuery(hqlSelect, Ticket.class);
selectQ.setParameter("st", "Confirmed");
List<Ticket> list = selectQ.list();

// DELETE query
String hqlDelete = "delete from Ticket where id = ?1";
Query deleteQ = session.createQuery(hqlDelete);
deleteQ.setParameter(1, 1);
int deleted = deleteQ.executeUpdate();

// Aggregate query
String hqlCount = "select count(*) from Ticket where status = 'Booked'";
Long count = (Long) session.createQuery(hqlCount).uniqueResult();
